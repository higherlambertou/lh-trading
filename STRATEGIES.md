# 策略說明手冊

本系統目前共有 **8 個策略**，分為兩類：

- **Tick 級策略**（逐筆報價觸發）：ma_cross、breakout、rsi、bollinger、momentum、scalp
- **K 棒級策略**（1 分 K 收完觸發）：orb、vwap_revert

同帳戶同時只能跑一個策略。

---

## 共用風控參數（所有策略都有）

| 參數 | 說明 | 預設 |
|------|------|------|
| `stop_loss_pts` | 固定停損點數（0=停用） | 0 |
| `take_profit_pts` | 固定停利點數（0=停用） | 0 |
| `daily_max_loss` | 當日最大虧損元，觸發後鎖開倉（0=停用） | 0 |
| `max_trades_per_day` | 當日最大進場次數（0=停用） | 0 |
| `trade_start_hhmm` | 可開倉起始時間，例如 `845`=08:45（0=不限） | 0 |
| `trade_end_hhmm` | 可開倉結束時間，例如 `1340`=13:40（0=不限） | 0 |

> 平倉單不受風控限制，停損停利隨時都能執行。

---

## 1. ma_cross — 均線交叉

**類型**：Tick 級 / 順勢

**邏輯**：
- 計算快線（`short_period` 筆均線）與慢線（`long_period` 筆均線）
- 快線**上穿**慢線 → 做多
- 快線**下穿**慢線 → 做空（同時反手平多）

**參數**：

| 參數 | 說明 | 預設 |
|------|------|------|
| `short_period` | 快線週期（tick 筆數） | 5 |
| `long_period` | 慢線週期（tick 筆數） | 20 |

**適合**：有趨勢的行情。震盪盤容易被反覆洗出。

**注意**：以 tick 筆數計算均線，日盤成交熱絡時 5 筆可能不到 1 秒，夜盤可能是幾分鐘，參數意義不穩定。

---

## 2. breakout — 區間突破

**類型**：Tick 級 / 順勢

**邏輯**：
- 記錄過去 `lookback` 筆的最高價與最低價
- 當前價格**突破最高價** → 做多
- 當前價格**跌破最低價** → 做空（同時反手平多）

**參數**：

| 參數 | 說明 | 預設 |
|------|------|------|
| `lookback` | 回顧 tick 筆數 | 20 |

**適合**：大波動突破行情。假突破多的盤面容易被套。

---

## 3. rsi — RSI 超買超賣

**類型**：Tick 級 / 逆勢（均值回歸）

**邏輯**：
- 計算 `period` 筆的 RSI（相對強弱指標）
- RSI 低於 `oversold`（超賣）→ 做多（預期反彈）
- RSI 高於 `overbought`（超買）→ 做空（預期回落）

**參數**：

| 參數 | 說明 | 預設 |
|------|------|------|
| `period` | RSI 計算週期 | 14 |
| `oversold` | 超賣門檻 | 30 |
| `overbought` | 超買門檻 | 70 |

**適合**：震盪盤。強趨勢行情中 RSI 會長期滯留超買/超賣區，逆勢進場容易套死。

**注意**：此 RSI 用簡單平均計算（非 Wilder 平滑），與一般看盤軟體的 RSI 略有差異。

---

## 4. bollinger — 布林通道

**類型**：Tick 級 / 逆勢（均值回歸）

**邏輯**：
- 中軌 = `period` 筆移動平均；上下軌 = 中軌 ± `num_std` 倍標準差
- 價格**跌破下軌** → 做多（預期回到中軌）
- 價格**突破上軌** → 做空（預期回到中軌）

**參數**：

| 參數 | 說明 | 預設 |
|------|------|------|
| `period` | 通道計算週期（tick 筆數） | 20 |
| `num_std` | 上下軌標準差倍數 | 2.0 |

**適合**：波動率穩定的震盪行情。單邊趨勢中逆勢進場會連續虧損。

---

## 5. momentum — 動能追漲殺跌

**類型**：Tick 級 / 順勢

**邏輯**：
- 比較當前價格與 `period` 筆之前的價格，計算漲跌幅
- 漲幅超過 `threshold_pct`% → 做多（追漲）
- 跌幅超過 `threshold_pct`% → 做空（殺跌）

**參數**：

| 參數 | 說明 | 預設 |
|------|------|------|
| `period` | 比較的回顧筆數 | 10 |
| `threshold_pct` | 觸發進場的漲跌幅門檻（%） | 0.1 |

**適合**：大行情爆發期。盤整期門檻難設，容易被噪音觸發。

---

## 6. scalp — 限價掃點

**類型**：Tick 級 / 短線

**邏輯**：
這是本系統最複雜的策略，有完整的狀態機：

1. **訊號判斷**（`idle` 狀態）：  
   - `momentum` 模式：統計最近 `momentum_window` 筆的外/內盤比例，超過 `momentum_threshold` → 觸發方向  
   - `random` 模式：隨機決定方向（測試用）

2. **掛限價單**（`pending` 狀態）：  
   依方向和 `entry_offset` 偏移掛 ROD 限價單；超過 `cancel_after_ticks` 個 tick 未成交自動取消

3. **持倉中**（`holding` 狀態）：  
   成交後自動掛 ROD 停利單（距進場價 `tp_pts` 點）；同時每 tick 檢查是否虧損超過 `sl_pts` → 市價停損

4. **冷卻**（`cooldown` 狀態）：  
   出場後等待 `cooldown_ticks` 個 tick 再重新找訊號

**參數**：

| 參數 | 說明 | 預設 |
|------|------|------|
| `tp_pts` | 停利點數 | 20 |
| `sl_pts` | 停損點數 | 60 |
| `entry_offset` | 掛單偏移（0=市價 正=被動 負=追價） | 0 |
| `cancel_after_ticks` | 掛單幾個 tick 後取消 | 15 |
| `momentum_window` | 外/內盤統計視窗（tick 筆數） | 20 |
| `momentum_threshold` | 觸發所需外/內盤比例（0~1） | 0.65 |
| `signal_mode_int` | 訊號模式（0=動量 1=隨機） | 0 |
| `cooldown_ticks` | 出場後冷卻 tick 數 | 30 |
| `max_qty` | 單次最多幾口 | 1 |

**適合**：日盤成交活躍時段（09:00-12:00）。外/內盤比例在成交量稀少的夜盤意義不大。

## 7. orb — 開盤區間突破

**類型**：K 棒級（1 分 K）/ 順勢

**邏輯**：
1. 開盤後前 `or_minutes` 分鐘（從 `or_start_hhmm` 起算）只觀察，記錄這段時間的最高價和最低價，形成「開盤區間」
2. 區間結束後，K 棒收盤價突破：
   - **高點 + `buffer_pts`** → 做多
   - **低點 - `buffer_pts`** → 做空
3. 每個方向每天最多進場一次（失敗不再追）
4. 出場由停損停利或 ATR 移動停損負責

**參數**：

| 參數 | 說明 | 預設 |
|------|------|------|
| `or_start_hhmm` | 觀察起點 HHMM（日盤開盤）| 845 |
| `or_minutes` | 觀察幾分鐘 | 15 |
| `buffer_pts` | 突破緩衝點數（防假突破） | 5 |
| `atr_period` | ATR 週期（分鐘K棒數） | 14 |
| `atr_trail_mult` | ATR 移動停損倍數（0=停用） | 0 |

**適合**：趨勢日（台指每周都有 2-3 天有明確方向）。

**建議參數**：
```
or_start_hhmm=845, or_minutes=15, buffer_pts=5
stop_loss_pts=60, max_trades_per_day=2
trade_end_hhmm=1330   # 尾盤不追
atr_trail_mult=2       # 啟用 ATR 移動停損，趨勢日吃更多
```

## 8. vwap_revert — VWAP 偏離回歸

**類型**：K 棒級（1 分 K）/ 逆勢（均值回歸）

**邏輯**：
- 每天開盤重置，累積計算當日 **VWAP（成交量加權均價）**
  - 典型價 = (最高 + 最低 + 收盤) / 3
  - VWAP = Σ(典型價 × 成交量) / Σ成交量
- 當前價格偏離 VWAP 超過 `dev_pts` 點：
  - **高於 VWAP + dev_pts** → 做空（預期回歸）
  - **低於 VWAP - dev_pts** → 做多（預期回歸）
- 價格回到 VWAP 附近（`exit_ratio × dev_pts` 以內）→ 獲利平倉
- 暖機 `warmup_bars` 根 K 棒後才開始交易（VWAP 初期樣本太少不穩定）

**參數**：

| 參數 | 說明 | 預設 |
|------|------|------|
| `dev_pts` | 觸發進場的偏離點數 | 40 |
| `exit_ratio` | 回歸多少比例平倉（×dev_pts） | 0.25 |
| `warmup_bars` | 開盤暖機 K 棒數 | 10 |
| `atr_period` | ATR 週期 | 14 |
| `atr_trail_mult` | ATR 移動停損倍數（0=停用） | 0 |

**適合**：震盪盤、開盤後急衝又回來的行情。強趨勢日逆勢連挨打，**務必設 `stop_loss_pts` 和 `daily_max_loss`**。

**建議參數**：
```
dev_pts=40, exit_ratio=0.25, warmup_bars=10
stop_loss_pts=60, daily_max_loss=3000, max_trades_per_day=5
```

## ATR 移動停損（orb / vwap_revert 專屬）

K 棒策略（orb、vwap_revert）有內建 **ATR 移動停損**：

- **ATR（平均真實波幅）**：用 Wilder 平滑法計算，反映近期波動幅度
- `atr_trail_mult > 0` 啟用：停損線 = 當前收盤價 - ATR × 倍數（多單）
- 多單的停損線**只升不降**；空單**只降不升**（移動方向跟著行情走）
- K 棒收盤價穿越停損線 → 市價出場

**優點**：
- 波動大時停損自動放寬（不被正常回檔洗掉）
- 波動小時自動收緊（鎖住更多獲利）

**建議**：`atr_trail_mult=2~3`（太小容易被雜訊洗、太大停損太寬）

## 選哪個策略？

| 市況 | 推薦策略 |
|------|---------|
| 今天有明確趨勢 | **orb**（方向確認後追） |
| 今天高低來回震盪 | **vwap_revert**（偏離回歸） |
| 開盤急衝型（10 分鐘決勝負） | **orb**（縮短 or_minutes） |
| 想測試系統下單是否正常 | **scalp**（signal_mode=random，小 sl_pts） |
| 還不確定 | **orb** 加 `stop_loss_pts=60`，先觀察行為 |

## 常見操作錯誤

1. **兩個策略同時跑** → 系統會拒絕（回傳 409），不是 bug
2. **停損沒設** → 策略虧損不止，強烈建議一定要設 `stop_loss_pts`
3. **vwap_revert 遇到趨勢日** → 逆勢一路虧，要有 `daily_max_loss` 保護
4. **orb 不等到區間結束就想看結果** → 區間時間內什麼都不會發生，正常
5. **scalp 信號模式設成 random** → 純隨機方向，只拿來測試系統，不要真錢跑
